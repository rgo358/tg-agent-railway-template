## ⚙️ ТЕХНИЧЕСКИЕ ДЕТАЛИ

### Архитектура
```
Telegram (ты) 
    ↓
Telethon Client (слушает сообщения)
    ↓
    ├→ Если "обнови" → index_3months() 
    │   ├→ Скачивает сообщения за 90 дней
    │   ├→ Фильтрует короткие сообщения
    │   ├→ Кодирует в embeddings (SentenceTransformers)
    │   └→ Сохраняет в ChromaDB
    │
    └→ Иначе → analyze(text)
        ├→ Ищет 40 похожих сообщений в БД (vector search)
        ├→ Берет контекст найденных сообщений
        ├→ Отправляет в Groq API
        └→ Возвращает ответ в Telegram
```

### Технологии
- **Telethon** - Telegram клиент (Python)
- **ChromaDB** - векторная БД (embeddings)
- **SentenceTransformers** - многоязычные embeddings
- **Groq API** - LLM для анализа (mixtral-8x7b)
- **Docker** - контейнеризация для Railway

### Ограничения
- Индексирует последние **30 дней** (не больше)
- Groq: **30 запросов в минуту** (бесплатно)
- ChromaDB: **локальная БД** (теряется при перезапуске на Railway - но быстро переиндексируется)
