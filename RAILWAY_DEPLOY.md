## 🎯 Способ развертывания на Railway

### **Вариант 1: One-Click Deploy (РЕКОМЕНДУЕТСЯ) ⭐⭐⭐**

Нажми эту ссылку:
```
https://railway.app/new/template?template=https%3A%2F%2Fgithub.com%2Frgo358%2Ftg-agent-railway-template
```

**Это автоматически:**
1. Создаст новый проект на Railway
2. Прочитает `railway.json` конфигурацию 
3. Покажет форму для ввода переменных
4. Раздеплоит контейнер с твоего GitHub репо

**Переменные для ввода:**
```
SESSION_STRING=(получи через get_session.py)

API_ID=(из https://my.telegram.org/apps)

API_HASH=(из https://my.telegram.org/apps)

XAI_API_KEY=(из https://console.x.ai)
```

Где получить каждую переменную:
1. **SESSION_STRING** → Запусти локально: `python get_session.py` (требует авторизацию в Telegram)
2. **API_ID** и **API_HASH** → https://my.telegram.org/apps (создай или используй приложение)
3. **XAI_API_KEY** → https://console.x.ai (создай API ключ, начнется с `xai-`)

6. **Deploy** ✅

## 🔍 Что происходит при развертывании

### Railway прочитает `railway.json`:
```json
{
  "build": {
    "builder": "DOCKERFILE",
    "dockerfilePath": "Dockerfile"
  },
  "deploy": {
    "startCommand": "python tg_super_agent.py",
    "restartPolicyType": "ON_FAILURE",
    "restartPolicyMaxRetries": 10
  }
}
```

### Процесс:
1. Railway загружает репозиторий
2. Собирает Docker образ (на основе `Dockerfile`)
3. Создает контейнер с твоим Python приложением
4. Запускает `python tg_super_agent.py`
5. Автоматически перезагружает при ошибках (до 10 раз)

### Что делает твой агент:
1. **Подключается** к Telegram через SESSION_STRING
2. **Индексирует** последние 3 месяца сообщений из всех чатов
3. **Векторизирует** с помощью многоязычной модели
4. **Сохраняет** embeddings в ChromaDB (локальная БД в контейнере)
5. **Слушает** новые сообщения в "Сохраненные сообщения" Telegram
6. **Отвечает** на запросы используя Grok API через поиск похожих сообщений

## 🛠 Если что-то не работает

### **Ошибка: "Unauthorized" при развертывании**
- Проверь что репозиторий публичный
- Заново авторизуйся в GitHub на Railway

### **Ошибка: "Failed to send telemetry"**
- Это НЕ ошибка! Просто предупреждение от ChromaDB
- Агент все равно работает корректно

### **SESSION_STRING истек**
- Создай новый используя `get_session.py`:
  ```bash
  python get_session.py
  ```
- Обнови переменную в Railway Dashboard

### **Контейнер постоянно перезагружается**
- Проверь логи для деталей
- Убедись что все 4 переменные установлены
